# بطاقات NFC للمكفوفين

نسخة أولية عملية لمشروع يعتمد على `NFC Card 13.56MHz` لفتح صفحة تعريف فورية عند لمس الكارت بالموبايل.

## الفكرة

- الكارت لا يخزن كل البيانات.
- يتم تخزين رابط فقط مثل: `https://your-domain.com/p/card-001`
- عند فتح الرابط تظهر صفحة فيها:
  - الاسم
  - العنوان
  - رقم الموبايل
  - رقم الطوارئ
  - وصف الحالة
  - تسجيل صوتي يشرح التشخيص أو أي تعليمات مهمة

## الموجود في النسخة الحالية

- صفحة رئيسية تعرض فكرة المشروع والبطاقات الحالية
- صفحة عامة لكل بطاقة على المسار `/p/:publicCode`
- لوحة إدارة على `/admin`
- إضافة وتعديل وحذف بطاقات
- رفع ملف صوتي لكل حالة
- قاعدة بيانات محلية `SQLite`

## تشغيل المشروع

```bash
npm install
npm start
```

لو عايز الرابط الظاهر داخل المشروع يكون رابطًا حقيقيًا للكارت بدل `localhost`، شغّل المشروع هكذا:

```powershell
$env:PUBLIC_BASE_URL="http://192.168.1.68:3000"
npm start
```

أو استخدم دومينك الحقيقي بدل عنوان الشبكة الداخلية.

بعد التشغيل:

- الصفحة الرئيسية: `http://localhost:3000`
- لوحة الإدارة: `http://localhost:3000/admin`
- بطاقة تجريبية: `http://localhost:3000/p/demo-card`

## كتابة الرابط على كارت NFC

اكتب الرابط العام فقط على الكارت، مثل:

```text
http://YOUR-IP:3000/p/card-001
```

مهم: لا تكتب `http://localhost:3000/...` على الكارت، لأن الهاتف سيعتبر `localhost` هو الهاتف نفسه وليس الكمبيوتر أو السيرفر.

يمكن كتابة نفس الرابط أيضًا داخل `QR Code` كخطة بديلة لو جهاز ما قرأش الـ NFC.

## ملاحظات مهمة قبل الاستخدام الفعلي

- النسخة الحالية فيها تسجيل دخول بسيط للإدارة باستخدام `ADMIN_USERNAME` و `ADMIN_PASSWORD`.
- يفضل عدم عرض بيانات طبية حساسة للعامة.
- من الأفضل لاحقًا إضافة:
  - صلاحيات مستخدمين
  - سجل عمليات
  - استضافة آمنة مع دومين وشهادة SSL
  - ربط خرائط أو واتساب إذا احتجت

## تشغيله أونلاين على Render

هذه النسخة تعتمد على:

- `SQLite`
- رفع ملفات صوتية إلى القرص المحلي

لذلك تحتاج خدمة ويب عادية مع `persistent disk`، وليست استضافة serverless.

### لماذا اخترنا Render لهذا المشروع

- يدعم تطبيقات `Node.js + Express` مباشرة
- يوفر رابط HTTPS جاهز
- يدعم `persistent disk` لحفظ قاعدة البيانات والملفات الصوتية

### ملفات النشر الجاهزة داخل المشروع

- `render.yaml`
- `.node-version`
- `.env.example`

### خطوات النشر

1. ارفع المشروع إلى GitHub.
2. افتح Render وأنشئ خدمة جديدة من نوع `Web Service`.
3. اربط المستودع.
4. Render سيقرأ ملف `render.yaml` تلقائيًا.
5. عند إنشاء الخدمة، أدخل قيمة `ADMIN_PASSWORD` عندما يطلبها Render.
6. بعد انتهاء النشر، سيفتح الموقع على رابط شبيه بـ:

```text
https://your-app.onrender.com
```

### مهم جدًا

- لا تستخدم الخطة المجانية إذا كنت تريد الاحتفاظ بقاعدة البيانات والملفات المرفوعة.
- هذه البنية مناسبة كـ MVP أو نسخة أولية عملية.
- لو أردت قابلية توسع أفضل لاحقًا، انقل البيانات إلى `Postgres` والملفات إلى `S3` أو `Supabase Storage`.
